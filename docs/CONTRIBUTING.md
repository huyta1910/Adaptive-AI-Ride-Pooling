# Contributing

## Branch Strategy

- `main`: stable release branch
- `develop`: integration branch for hackathon work
- `feature/frontend-passenger`: passenger-facing frontend work
- `feature/frontend-driver`: driver-facing frontend work
- `feature/backend`: backend API and data work
- `feature/ai`: AI integration work

## Protected Areas

Feature branches must not modify:

- layouts
- shared components
- theme
- routing
- config

Feature teams should add new files under feature-specific folders and keep shared changes behind explicit team agreement.
