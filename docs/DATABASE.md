# Database

The database foundation uses PostgreSQL, SQLAlchemy models, and Alembic migrations.

## Tables

- `users`
- `drivers`
- `passengers`
- `bookings`
- `ride_pool_groups`
- `ride_pool_members`
- `weather_events`
- `notifications`
- `trip_history`

The schema is intentionally neutral and contains no business workflow logic.
